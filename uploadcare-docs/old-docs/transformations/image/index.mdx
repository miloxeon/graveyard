# Image Transformations

With Uploadcare, you can easily build custom image transformation workflows and
automate most of image manipulation and optimization tasks. For example, you can
set up a chain of actions for user-generated images that'll unify their look.

Every URL transformation generates a modified image version on the fly, while
the original file stays intact. The transformed image version will be cached on
[CDN nodes][delivery-cdn] to optimize delivery.

You can stack multiple transformations (e.g.,
[overlay the source image][transformations-image-overlay-self]).
Uploadcare supports virtually any image input format. The processed output
formats are distilled down to: `PNG`, `JPEG` or `WebP`.

Also, you can use [Proxy][delivery-proxy] to apply image transformations without
explicitly uploading them:

 ```
 https://yoursite.ucr.io/-/preview/-/resize/500x/https://yoursite.com/assets/image.jpg
 ```

## Limitations
{id="limits"}

This section describes the limits to consider when implementing Uploadcare
Image Transformations.

### Core Operations
{id="core-operations"}

When applying image transformations, add at least one of the following
operations via their respective URL directives:

* [`resize`][transformations-image-resize]
* [`scale_crop`][transformations-image-scale-crop]
* [`preview`][transformations-image-preview]

### Output Image Dimensions
{id="dimensions"}

The dimensions you specify for the last operation should not exceed 3000x3000
pixels. You can go up to 5000x5000 pixels by explicitly setting
your [image format][transformations-image-format] to JPEG, `cdn›/format/jpeg/`.

You can extend the output max resolution up to 8192x8192, and up to 16384x16384
for JPEGs in trade-off for increased latency. [Contact support][mail-help] to learn more
about this option.

### SVG files

Any image transformation CDN URL is valid with an SVG file. But it doesn't
affect it in any way and it always returns untouched SVG body. We’ll be adding
basic resize and crop operations support for SVG images later.

*SVGs uploaded before May 26, 2021 still have `is_image: false` and adding
processing operations to them will result in error. Contact
[support][mail-help] to batch process previously uploaded files.*

### Image Resolution
{id="size-limit"}

We don’t provide on-the-fly image transformations for images greater than
75 Mpx in resolution: those are tagged as **non-image** files and can only be delivered
via CDN as-is. Adding processing operations to non-image files will result
in an error.

### Simple Rotation
{id="simple-rotation"}

When the only image transformation you want is rotating, consider using the
[`preview`][transformations-image-preview] control without any arguments. When
you use any of the transformations, we automatically rotate images according to
their EXIF orientation.

### Animated Images
{id="animated"}

Animated images are treated as static by the transformations engine, consider
checking out our [GIF to Video][transformations-image-gif-to-video] workflow
optimized for animated images delivery.
